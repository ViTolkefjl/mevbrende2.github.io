# Мы в бренде — сайт

Статический сайт байерского сервиса «Мы в бренде»: лендинг + страницы документов
(оферта, политика конфиденциальности, пользовательское соглашение).

## Чек-лист перед запуском

### 1. Заполнить реквизиты ИП и e-mail (обязательно!)

Без реальных реквизитов сайт публиковать нельзя (ст. 8–9 Закона о защите прав потребителей).
Все места помечены комментариями `TODO` — найти их можно поиском по слову `TODO`:

- `index.html` — подвал: ФИО ИП, ИНН, ОГРНИП, e-mail (два места: карточка «Почта» и подвал);
- `oferta.html` — раздел «1. Предмет оферты»: ИНН и ОГРНИП.

E-mail `info@mev-brende.ru` — заглушка: замените на реальный адрес или заведите такой ящик.

### 2. Подставить домен

Сейчас в разметке прописан адрес GitVerse Pages: `vitol.gitverse.site/mevbrende`.
После покупки домена выполните в папке сайта одну команду (macOS/Linux),
подставив свой домен вместо `ваш-домен.ru`:

```bash
LC_ALL=C sed -i '' 's|vitol.gitverse.site/mevbrende|ваш-домен.ru|g' index.html oferta.html privacy.html agreement.html robots.txt sitemap.xml
```

Или вручную замените `vitol.gitverse.site/mevbrende` на свой домен в этих же шести файлах.

### 3. Опубликовать на GitVerse Pages

1. Создайте **публичный** репозиторий на [GitVerse](https://gitverse.ru) (название без точки в начале).
2. Запушьте содержимое этой папки в ветку `main`.
3. В настройках репозитория включите **CI/CD**, затем включите **Pages** (источник — ветка `main`, корень `/`).
4. Сайт появится по адресу `https://ваш-ник.gitverse.site/название-репозитория`.
5. Прикрепите купленный домен в настройках Pages.

### 4. После запуска (по юридическому аудиту)

- Проверить, подано ли уведомление в Роскомнадзор о обработке персональных данных.
- Финальные тексты оферты и политики согласовать с юристом.

## Структура

```
index.html      — лендинг
oferta.html     — публичная оферта
privacy.html    — политика конфиденциальности
agreement.html  — пользовательское соглашение
styles.css      — общие стили
fonts/          — шрифты (self-hosted, без внешних запросов)
favicon.svg     — иконка сайта
og-image.png    — превью для соцсетей (1200×630)
robots.txt, sitemap.xml — для поисковиков
```
